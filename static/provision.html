User Provisioning

Resource server API for user management

Resource Server Credentials

Find or create a user by username and/or email. If the user exists, their info is returned. If not, a new account is created with an invitation token for setting up a password.

At least one of username or email is required. If both match different existing users, an ambiguous-match error is returned.

Look up an existing user by user ID, username, or email. Read-only — does not create users or generate invitations.

User ID is the preferred lookup key for post-invitation operations.

Link a user to a client application. Idempotent — linking an already-linked user succeeds silently. The client must belong to this resource server.

Remove a user's link to a client application. Idempotent — unlinking a non-linked user succeeds silently.

List all users linked to a client application. Returns paginated results with user IDs, usernames, and active status.
